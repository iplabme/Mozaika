# Визуальный профиль PPTX для сценария поиска инсайтов

Статус: обязательный дизайн-референс для последнего редактируемого PPTX в
`insight_deck`.

Источник владельца:
`references/scenario-insight-ds-role-analytics-reference.pptx`, SHA-256
`66430f7a96d66546eaa6c9fd8fad971e925a40c09ce2e2ae91047f16263a6c1d`.

## Граница заимствования

Референс задаёт только визуальную грамматику. Брать из него цвета,
типографическую иерархию, пропорции зон, правила выравнивания, оформление
графиков, таблиц, карточек, источников и управленческих выводов. Не копировать
темы, данные, формулировки, число слайдов, порядок слайдов или конкретный набор
композиций. Состав и последовательность нового PPTX выводить из выбранного
владельцем сторилайна и принятой HTML-презентации текущего запуска.

## Токены

- Формат: 16:9, 13,33 × 7,5 дюйма.
- Основной светлый холст: `#EFEBE4`; белая доказательная поверхность:
  `#FFFFFF`.
- Тёмный чернильно-синий: `#223240`; дополнительный тёмный: `#2C4452`.
- Основной зелёный: `#7AA88C`; тёмный зелёный: `#4A7A5E`.
- Категориальные акценты: синий `#889EB6`, горчичный `#C8A258`, лавандовый
  `#9E94C6`, коралловый `#C88A7C`.
- Линии и нейтральные границы: `#DCD6CC`.
- Семантические заливки: спокойный зелёный `#BDD8C6`, предупреждение
  `#ECDCAC`, дефицит `#ECCCC2`, критический дефицит `#F8CCC8`.
- Заголовки: `PT Sans Narrow`; основной текст: `PT Sans`. При отсутствии
  шрифтов использовать системные узкие/обычные sans-serif аналоги без изменения
  размеров и иерархии.

## Грамматика размещения

- Обложка может использовать тёмное поле и отдельную зелёную вертикальную
  плоскость; заголовок остаётся главным объектом, декоративные окружности не
  конкурируют с текстом.
- На аналитических слайдах использовать тонкую зелёную верхнюю линию, крупный
  тезис слева, короткое пояснение под ним и небольшой смысловой маркер справа.
- Основная доказательная зона занимает примерно две трети или три четверти
  ширины. Узкая правая колонка допустима только для интерпретации, легенды или
  управленческого следствия.
- Карточки строить открыто, с тонкими границами, одинаковыми базовыми линиями и
  одним семантическим цветом на сущность. Не превращать слайд в стену одинаковых
  серых панелей.
- Графики делать нативными и редактируемыми, с прямыми подписями, спокойной
  сеткой и единым масштабом сравнения. Источник, база и единица располагаются
  рядом с визуализацией.
- Таблицы использовать для gap-анализа и проверяемых деталей: тёмная шапка,
  ровные колонки, семантические заливки только по значению, а не для декора.
- Решения и действия можно собирать в две–три выровненные колонки с номером,
  владельцем, сроком и доказанным основанием; количество колонок определяется
  содержанием текущего сторилайна.
- Нижний источник и оговорки держать компактно на общей базовой линии. Не
  добавлять служебные идентификаторы, номера вариантов, хеши и имена контрактов.

## Применение

Использовать `presentation-skill` с пресетом `mozaika-insight`. Перед рендером
заменить примерный список слайдов в
`templates/scenario-insight-presentation-skill-outline.example.json` на
последовательность из принятой HTML-презентации. PPTX обязан сохранить те же
утверждения, числа, порядок смыслов и охват пользовательских вопросов, но может
выбирать подходящую композицию для каждого слайда из этой визуальной грамматики.
