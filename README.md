# Mozaika

Mozaika — расширение для ГигаАгента Ouroboros, которое превращает файлы, папки и URL с данными в проверяемые управленческие материалы: интерактивный дэшборд, варианты истории, HTML-презентацию, карточки спикера и редактируемый PPTX.

Система рассчитана на два режима работы:

- **поиск инсайтов** — агент исследует данные, показывает доказательства и 2–3 способа подачи, владелец выбирает стратегию, после чего Mozaika строит сторилайн и комплект презентационных материалов;
- **регулярный отчёт** — агент обновляет известный отчёт по свежей выгрузке, пересчитывает KPI, ищет аномалии и выпускает материалы по утверждённому шаблону без лишних согласований.

Mozaika не является автономным приложением и не дублирует агентную платформу. Очередь задач, длительное выполнение, проекты, память, контекст, инструменты, жизненный цикл скиллов, ревью, безопасность и наблюдаемость предоставляет Ouroboros.

## Быстрый запуск

### Требования

- Ouroboros `6.71.0` или совместимая версия с `PluginAPI 1.3`;
- настроенная модель и разрешённые инструменты Ouroboros;
- Python и Node.js из поставки Ouroboros для исполняемых скиллов;
- установленный и прошедший ревью пакет Mozaika.

### Установка в рабочую среду Ouroboros

1. Скопируйте `skills/mozaika` в `data/skills/external/mozaika` рабочей папки Ouroboros.
2. Скопируйте необходимые каталоги из `skills/` в `data/skills/external/`, не перезаписывая пользовательские артефакты и состояние скиллов.
3. Скопируйте `brandbook/` в `data/brandbook/mozaika/`.
4. В разделе скиллов Ouroboros выполните предварительную проверку, ревью, выдайте необходимые разрешения и включите `mozaika`.
5. Перезапустите Ouroboros и откройте страницу виджетов.
6. Выберите `Данные для поиска инсайтов` или `Данные для рутинного отчёта`, добавьте источники, уточните поручение и запустите задачу.

Пакеты скиллов и состояние выполнения разделены: проектный каталог — источник правды разработки, `data/state/skills/<name>` — рабочее состояние. Не копируйте токены, локальные окружения `.ouroboros_env` или содержимое `data/state` между машинами.

## Проверка

```bash
python3 -m pytest -q
```

Ожидаемый результат текущей версии:

```text
36 passed, 112 subtests passed
```

Тесты проверяют регистрацию расширения, оба виджета, смешанный ввод, безопасность путей и URL, сохранение задания, owner choice, контракты области и утверждений, брендбук, карточки спикера и запрет ложного статуса готовности.
