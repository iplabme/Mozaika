# Сценарий 2: «Пятничный автопилот»

Статус: утверждённый владельцем профиль процесса и дизайна для второго виджета Mozaika.

## Контур процесса

1. Получить Excel или совместимые табличные данные по работе команд и сохранить исходник без изменений.
2. Проверить схему, полноту, типы, дубли и период; по возможности сопоставить с прошлым принятым запуском.
3. Обновить управляемый HTML-dashboard по шаблону `templates/scenario-2-dashboard.template.html`.
4. После расчёта dashboard отдельно проверить аномалии и важные моменты относительно истории, целей и соседних сегментов; не принимать сам факт экстремума за управленческий риск без контекста.
5. Собрать богатую HTML-презентацию по `templates/scenario-2-presentation.template.html`, сохранив экранную логику: контекст → KPI → динамика → срез → кульминационная аномалия → полная таблица → действия.
6. Последним шагом собрать редактируемый PPTX только установленным и прошедшим review скиллом `presentation-skill`, используя `--style-preset mozaika-weekly` и outline на основе `templates/scenario-2-presentation-skill-outline.example.json`.
7. Сверить общие цифры и выводы между dashboard, HTML и PPTX, отрендерить PPTX и проверить каждый слайд на наложения, выходы за границы и читаемость.

Обычный запуск не содержит выбора storyline и не создаёт storytelling cards: процесс и порядок рассказа уже известны. Остановиться следует только при отсутствующем обязательном источнике, несовместимой схеме, конфликтующей метрике без правила приоритета или таком изменении данных, которое делает утверждённый шаблон вводящим в заблуждение.

## Дизайн dashboard

- Светлый тёплый холст, белые панели, тонкие бежевые границы, пастельные лавандовые, шалфейные, голубые, золотые и розовые акценты.
- Верхняя строка: название отчёта, период, срез и компактные элементы управления.
- Далее: четыре KPI, блок «требуют внимания», динамика, сравнение сегментов, структура работы и детальная таблица.
- Dashboard должен быть действительно управляемым: фильтр периода и среза, поиск и сортировка таблицы, фильтр важности, настройка плотности, скрытие второстепенного раздела и полный сброс.
- В dashboard нет загрузчика исходного Excel: виджет Mozaika уже принял и сохранил источник. Нет storytelling-карточек, служебных идентификаторов, путей, хешей или названий внутренних стадий.
- Внешние CDN запрещены. Графики строятся встроенным SVG/Canvas/JS или проверенным runtime, встроенным внутрь автономного HTML.

## Дизайн HTML-презентации

- Один главный вывод на экран, минимум декоративного текста, доказательства занимают основную площадь.
- Заголовки редакционные и спокойные; числовые карточки, графики и таблица повторяют визуальный язык dashboard.
- Кульминационный экран посвящён главной аномалии с фактом, нормой, отклонением и управленческим следствием.
- Обязательны плавные переходы, клавиатурная навигация, полноэкранный и обзорный режим, печать, адаптивность и `prefers-reduced-motion`.
- HTML остаётся автономным; служебная подпись renderer из референса не переносится.

## Дизайн PPTX

PPTX является последним, дополнительным редактируемым форматом, а не заменой HTML. Его создаёт `presentation-skill`; `anthropic-pptx` и скилл с идентификатором `pptx` не используются. Авторитетная структура и цифры берутся из принятого HTML-outline и claim registry. Для визуальной близости использовать стиль `mozaika-weekly`: тёплый фон `F4F1EC`, белые поверхности, текст `3D3A42`, лавандовый `B8AEE0`, шалфейный `A8C6A1`, розовый сигнал `DFA5AE`, границы `EAE3D9`, Georgia для заголовков и Segoe UI/Calibri для основного текста.
