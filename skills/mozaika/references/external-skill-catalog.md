# Каталог внешних скиллов Mozaika

Каталог описывает весь установленный пул `data/skills/external`. Он помогает
выбирать инструмент осознанно, но не заменяет `list_skills`, полное чтение
`SKILL.md`, проверку свежести ревью, зависимостей, разрешений и точного entry
перед каждым этапом.

Подробные триггеры, связки и требования к результатам девяти переданных
`anthropic-*` скиллов находятся в `references/anthropic-skill-routing.md`.

## Приоритетная установленная группа Anthropic

Это все девять скиллов из `new-skills/ALL_SKILLS_BUNDLE.md`. Они восстановлены
с исходными файлами и единообразно установлены под идентификаторами
`anthropic-*`. До каждого релевантного этапа Mozaika обязана сначала проверить
именно эту группу, полностью прочитать `SKILL.md` подходящих кандидатов и выбрать
подходящий Anthropic-скилл, даже если его инструкция короткая. Отступление
допустимо только при доказанной несовместимости с контрактом, безопасностью,
runtime или качеством; причина фиксируется в `skill-selection.json`.

| Скилл | Для чего выбирать | Важные ограничения |
|---|---|---|
| `anthropic-data-validation` | Проверка методологии, точности, смещений, полноты и готовности анализа к показу руководителю. | Это контроль качества, а не самостоятельный движок загрузки или рендера. |
| `anthropic-data-visualization` | Выбор типа графика, визуальное кодирование и создание аналитических визуализаций. | Перед выбором проверить доступность Python-библиотек и формат итогового артефакта. |
| `anthropic-interactive-dashboard-builder` | Автономные интерактивные HTML-дашборды с фильтрами и Chart.js; сильный кандидат и для HTML-презентации. | Проверить автономность файла, доступность офлайн-ресурсов и визуальный QA. |
| `anthropic-performance-analytics` | Маркетинговые KPI, тренды, сравнения кампаний и рекомендации по оптимизации. | Использовать только когда предметная область действительно маркетинговая. |
| `anthropic-sql-queries` | Проектирование и проверка SQL для основных хранилищ и диалектов. | Сам по себе не даёт доказательство исполнения запроса и не расширяет доступ к БД. |
| `anthropic-statistical-analysis` | Описательная статистика, тренды, выбросы, гипотезы и значимость. | Требовать зафиксированные допущения, размер выборки и ограничения вывода. |
| `anthropic-pptx` | Оригинальный набор инструкций и утилит Anthropic для PowerPoint. | Сохранён целиком, но запрещён в Mozaika; финальный редактируемый PPTX создаёт только `presentation-skill`. |
| `anthropic-dashboard-architect` | Премиальный single-file HTML dashboard, интервью об аудитории, поиск аномалий, KPI, heatmap, funnel и attention cards. | Для уже поставленной задачи не повторять вопросы, ответы на которые очевидны из handoff; проверять CDN/offline и доступность. |
| `anthropic-snowflake-semanticview` | Создание, изменение и проверка Snowflake semantic views через Snowflake CLI. | Любое чтение или изменение Snowflake требует явного полномочия владельца и настроенного соединения. |

## Роли и адаптеры Mozaika

| Скилл | Для чего выбирать | Важные ограничения |
|---|---|---|
| `mozaika` | Оркестрация сценариев, классификация решений, контракты, сохранение артефактов и контроль этапов. | Не подменяет специализированные скиллы этапов. |
| `analyze-report-data` | Ответственность data-роли: инвентаризация, профиль, обратимая очистка, KPI, claims и lineage. | Выбирает исполнительные скиллы динамически; не должен создавать непроверенный одноразовый обход. |
| `build-insight-dashboard` | Ответственность dashboard-роли: графики, карточки, связь claim → chart и варианты подачи. | Не меняет молча входные данные и не выбирает сюжет за владельца. |
| `design-executive-storyline` | Ответственность storyline-роли: главный тезис, логика рассказа, последовательность и намерение экранов. | История строится до визуальной презентации и только по подтверждённым claims. |
| `build-routine-report` | Сквозной регулярный сценарий data → dashboard → HTML presentation → editable PPTX. | Формат и обучение рутины переиспользуются только после проверки нового среза. |
| `huggingface-datasets` | Полная инвентаризация публичного Hugging Face dataset repo, файлов, split/subset и revision. | Нулевой или подозрительно малый охват блокирует следующий этап до перепроверки. |
| `duckdb-analytics` | Воспроизводимый профиль и read-only SQL по CSV/TSV/JSON/JSONL/Parquet. | До запуска сверить CLI/JSON-контракт; структурные ошибки требуют смены подхода, а не серии повторов. |
| `antv-g2-dashboard` | Самодостаточный executive HTML-дашборд из типизированной спецификации AntV G2: связанные фильтры, перерисовка графиков, обязательные таблицы с поиском и сортировкой, настройка видимости разделов и плотности, общий сброс. | Вход нужно валидировать до исполнения и обязательно провести browser QA фильтров, таблиц, настройки вида, пустых состояний и сброса. Другой renderer нужен только для функций сверх контракта, например редактирования самих определений графиков или постоянного сохранения пользовательского layout. |

## Назначенный рендерер HTML-презентаций

| Скилл | Для чего выбирать | Важные ограничения |
|---|---|---|
| `html-presentation-studio` | Первый выбор владельца для финальной HTML-презентации: narrative spine, assertion titles, выразительная арт-дирекция, автономный vanilla HTML, fullscreen/overview, статический и Chromium QA. | Запускать через `skill_exec`; выходы только в append-only state с новым `--job-id`. Для Mozaika использовать self-contained vanilla или bespoke HTML, не CDN-вариант reveal.js. |

## Другие установленные внешние скиллы

| Скилл | Для чего выбирать | Важные ограничения |
|---|---|---|
| `executive-storytelling` | Методика управленческого повествования: governing thought, tension, Three Things, ask. | Методика поддерживает storyline-роль, но не получает право менять доказательства. |
| `presentation-skill` | Генератор редактируемых PowerPoint с нативными диаграммами и таблицами. | Обязательный последний renderer всех сценариев: `mozaika-weekly` для routine/weekly и `mozaika-insight` для insight после принятого HTML и карточек спикера. |

## Правило выбора

1. Отфильтровать несовместимые и неготовые скиллы.
2. Для presentation-этапа сначала оценить назначенный владельцем
   `html-presentation-studio`; для остальных этапов сначала оценить установленную
   Anthropic-группу.
3. Сравнить её с остальным установленным пулом по качеству, проверяемости и
   стоимости исполнения; при равенстве выбрать Anthropic.
4. Для HTML-презентации выбрать `html-presentation-studio`,
   если его live-состояние готово и контракт совместим. Минимум:
   один автономный файл, экранная навигация, клавиатура, полноэкранный и обзорный
   режимы, адаптивность, интерактивные графики, доступность, reduced motion,
   печать и визуальная проверка в браузере.
5. Для последнего PPTX-шага выбрать только `presentation-skill`: пресет
   `mozaika-weekly` для `routine_report`/`weekly_autopilot`, пресет
   `mozaika-insight` для `insight_deck`. Во втором случае референс задаёт только
   визуальную грамматику, не структуру колоды. `pptx` и `anthropic-pptx` не
   являются кандидатами.
6. Искать новый скилл в сети только если установленный пул не закрывает контракт.
   Постоянная установка, зависимости, разрешения и включение требуют
   `owner_choice`.

`anthropic-pptx` запрещён для любого шага Mozaika. `presentation-skill` допустим
только для выделенной роли `pptx` в двух фиксированных сценариях. Старые каталоги
без префикса `anthropic-` не должны появляться в selection receipts.
